1. Project Overview
1.1 Purpose
A web-based self-service Point-of-Sale (POS) and inventory system for DICT Region 2 staff. Staff browse goods, add to cart, and checkout via Cash or Debt (utang). Admins manage products, stock, payments, and audit the debt ledger. The system replaces manual tally sheets with a transparent, immutable digital record.
1.2 Goals
Mobile-first self-service; P95 page load <2 s on 4G.
PocketBase as single API layer; MySQL 8 InnoDB as durable data store.
Atomic stock decrement via PocketBase hooks — stock never goes below zero.
Immutable debt ledger; balance computed from entries, never stored as a column.
CSV exports for COA accountability.
Deployable on Hostinger KVM2 via Docker Compose in a single command.

2.2 PocketBase + MySQL — How They Connect
PocketBase natively uses SQLite. To use MySQL 8 the binary must be built from source with the MySQL driver. This is done once and the resulting binary is baked into the Docker image.
PocketBase reads DB_DSN on startup and connects to MySQL via tcp.
All pb_migrations/*.js files run automatically in alphabetical order on first boot.
pb_hooks/*.js files implement custom business logic (checkout, debt payments, audit logging).
PocketBase file storage (product images) is mapped to ./pb_data volume.

5.3 POST /api/checkout (Custom Hook)
Hook transaction steps (all or nothing):
SELECT products FOR UPDATE where id IN (item ids) — acquires InnoDB row-level locks.
Validate stockQty >= requested qty for each item. Throw 409 if insufficient.
If paymentType=debt AND debtCeiling=0 → throw 402 "not allowed".
If paymentType=debt AND debtCeiling IS NULL → skip ceiling check (unlimited).
If paymentType=debt AND debtCeiling>0 → compute SUM(debt_ledger.amount) for user; if balance+total > ceiling → throw 402 with remaining.
Create order record.
Create order_items with unitPrice snapshot.
Decrement stockQty on each product.
Create stock_movements records (reason=sale, actorId=userId).
If paymentType=debt → create debt_ledger entry (type=purchase, amount=+total).
